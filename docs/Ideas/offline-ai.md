# Hypothesis: AI Service is only available during sessions or for batch jobs

The App should be usable without a GPU backed AI model. If it needs a very small model for certain decisions, that can run in CPU with decent performance that is fine. 

The app should be installable on a VPS server than runs 24/7 but has only access to an llm when a home gaming machine is booted up...

However that rules out LLM backed Chat experience for the control of the sub during normal time. We have to think about alternatives...

Ideas:

Traditional chat-bot experience with decision trees and prefabricated answers. These could be batch generated by an LLM based on the character and goal parameters.
Same for tasks, they are batch generated as well.

Real LLM Experience only during active sessions.

The denial timer concept has to be reiterated into a full fledged punishment system.
